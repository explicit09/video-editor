# Organization Knowledge

## Rules

### Rule: Process is Everything
- **Applies to:** All workflows
- **Rule:** Consistent workflow determines faster output and better decision-making. Develop repeatable processes for common content types and stick to them.
- **Reasoning:** Standardized workflows reduce cognitive load and decision fatigue. When you follow proven steps, you spend mental energy on creative choices rather than logistics.

### Rule: Keep Drafts of Everything
- **Applies to:** All workflows
- **Rule:** Always maintain drafts and versions of completed work. Design your system to enable quick retrieval of previous iterations, cuts, or raw materials.
- **Reasoning:** You'll frequently need to reference earlier decisions, restore removed content, or repurpose material. Version control saves time and prevents redundant work.

### Rule: Color and Label Everything in Timeline
- **Applies to:** All workflows
- **Rule:** Use colors to visually represent clip status (rough cut, approved, color-graded, etc.). Apply consistent labeling conventions to all clips.
- **Reasoning:** Visual labeling eliminates mental parsing of clip names during fast-paced editing. Status colors at a glance tell you what work remains and what's complete.

### Rule: Name Clips Clearly and Descriptively
- **Applies to:** All workflows
- **Rule:** Name clips with the person/subject, not just generic names like "Interview_02". Include identifying information that makes clips findable and understandable weeks later.
- **Reasoning:** Descriptive names reduce search time, prevent confusion between similar clips, and speed up collaboration. Your future self will thank you for specificity.

### Rule: Centralized Project Folder Structure
- **Applies to:** All workflows
- **Rule:** Keep all project files in the same folder for easy backup and sharing. Organize subdirectories consistently across projects (Assets, Sequences, Exports, Archives).
- **Reasoning:** Centralized structure simplifies backups, prevents lost files, and makes handoffs to collaborators straightforward. It also reduces path errors in reconnect media scenarios.

### Rule: The Split System for Voiceover Content
- **Applies to:** Voiceover + B-roll content
- **Rule:** Create three timelines: Selects timeline (curated B-roll only), VO timeline (voiceover with no B-roll—"radio edit"), Main Edit timeline (combine both). Process each independently.
- **Reasoning:** This workflow reduces cognitive load by isolating decisions. Trimming voiceover separately ensures tight pacing; reviewing B-roll separately ensures it serves the audio; combining them focuses on synchronization only.

### Rule: Radio Edit for Spoken Content
- **Applies to:** All spoken/interview content
- **Rule:** Create a "radio edit" by trimming all empty space and filler from voiceover tracks for tight review. Use this as your reference edit.
- **Reasoning:** A clean voiceover baseline removes ambiguity about pacing and content flow. It's faster to review and serves as the foundation for all subsequent creative decisions.

### Rule: Multi-Camera Workflow Sequence
- **Applies to:** Multi-camera productions
- **Rule:** Follow this workflow: import all angles → create multi-cam sequence → sync angles → switch cut in real-time. Don't manually sync or cut between angles.
- **Reasoning:** Multi-cam sequences handle sync automatically and allow real-time switching during playback. This approach is faster and produces more precise cuts than manual methods.

### Rule: Build Quick Technique Repertoire
- **Applies to:** All workflows
- **Rule:** Develop a personal library of proven techniques for common scenarios (color grades, transitions, text animations, sound design approaches). Reuse before reinventing.
- **Reasoning:** A technique library accelerates execution and ensures consistency. You spend less time problem-solving and more time applying proven solutions.

### Rule: Quick First Look, Then Loop and Refine
- **Applies to:** Color grading and all visual techniques
- **Rule:** Apply initial look to ALL shots quickly using a lut or preset, then refine each shot individually. Don't perfect one shot before moving to the next.
- **Reasoning:** Quick application across all footage establishes visual consistency faster. Refinement loops are more efficient when you have baseline reference across all material.

### Rule: Ruthless Prioritization Under Time Constraints
- **Applies to:** Commercial/deadline-driven work
- **Rule:** In time-constrained sessions, prioritize ruthlessly. Identify critical moments, finish those first, then fill remaining time with supporting work.
- **Reasoning:** Deadline-driven work requires triage. Focusing on highest-impact elements first ensures deliverables are complete if time runs out, rather than everything being 80% done.

### Rule: Simpler Tasks Drive Faster Workflow
- **Applies to:** All workflows
- **Rule:** Breaking complex work into smaller, focused tasks paradoxically speeds things up. Simpler mental tasks = faster workflow. Don't attempt everything at once.
- **Reasoning:** Task-switching and cognitive load slow execution. Isolating individual decisions (color first, then cuts, then sound) creates momentum and reduces errors.

### Rule: Remove Tools and Options from View
- **Applies to:** All workflows
- **Rule:** Hide or remove tools/UI elements from your workspace that aren't needed for the current task. Fewer visible options force focus and prevent distraction.
- **Reasoning:** Decision paralysis slows workflows. Hiding alternate tools keeps attention on the active task and prevents context-switching.

### Rule: Balance Footage Before Creative Looks
- **Applies to:** Color grading and all graded content
- **Rule:** Correct exposure, white balance, and levels (technical balance) before applying stylistic color grades or creative looks.
- **Reasoning:** Technical problems are different from creative choices. Fixing technical issues first prevents you from chasing aesthetic problems that are actually exposure or white balance issues.

### Rule: Delete Unused Audio Tracks
- **Applies to:** All workflows
- **Rule:** Remove unused or offline audio tracks from your timeline to maintain clean workflow. Keep only necessary tracks active.
- **Reasoning:** Cluttered timelines slow navigation, increase render times, and create opportunities for mistakes. Cleanliness enables focus.

### Rule: Isolate Technical From Creative Problems
- **Applies to:** All workflows
- **Rule:** When troubleshooting, clearly separate technical problems (codec issues, file corruption, playback errors) from creative problems (pacing, color, composition). Apply technical solutions to technical problems; creative solutions to creative problems.
- **Reasoning:** Misdiagnosing the problem type wastes time. A pacing issue won't be solved by adjusting codecs; a sync issue won't be solved by changing color grades.

## Tips

1. **Create a template project** - Build a fully organized template with folder structure, color labels, naming conventions, and sample sequences. Duplicate it for each new project to ensure consistency from the start.

2. **Name clips with timecode or segment** - Include shoot date and segment type in clip names (e.g., "2026-04-08_Interview_Sarah_Q1_TechBackground"). This makes searching and organizing intuitive.

3. **Use bins or folders aggressively** - Don't dump everything in one media folder. Create subfolders for each scene, camera angle, or interview subject. This speeds up navigation and prevents media management errors.

4. **Lock the VO timeline while editing B-roll** - When using the Split System, lock the voiceover timeline to prevent accidental changes while focusing on B-roll placement. Unlock for final review.

5. **Archive old sequences** - Move completed or abandoned sequences to an "Archive" bin. This keeps your active timeline list short and prevents accidental edits to old versions.
