Thumbnail Renderer — Design Proposal

Programmatic thumbnail generation with real host photos, no AI image generation

Architecture

A new ThumbnailRenderer class in the EditorCore package that uses Core Graphics to compose thumbnails from real assets. Replaces AI image generation for the standard thumbnail workflow. AI providers (FLUX, Gemini) remain available for creative/experimental use.

Pipeline
1. Load host photos → 2. Remove backgrounds (Vision) → 3. Feather edges → 4. Draw background gradient → 5. Composite hosts → 6. Render text + logo + accents → PNG out

Components

ThumbnailRenderer
Main entry point. Takes a ThumbnailConfig, returns PNG Data. Orchestrates the pipeline.
ThumbnailConfig
Data struct: title, subtitle, layout, host photos, brand (colors, logos). Loaded from overlay template JSON.
ThumbnailLayout (enum)
splitPanel, centeredFaceOff, textHeavy — each knows how to position hosts + text in the canvas.
PhotoCutout (utility)
Uses existing PersonMasker + CIFilter gaussian blur on mask edges for feathered cutouts.

Key Decisions

Photos
Fixed host photos from Documents dir. Background removed via Vision framework. Feathered edges via gaussian blur on mask.
Title
Auto-generated from analyze_transcript, with manual override. Line 1 white, line 2 gold. CoreText rendering.
Brand
Colors + logos loaded from overlay template JSON (brand section). Technolgia Talks hardcoded for now, structured for future templates.
Output
1536×1024 PNG. Instant (no API calls). Saved to Thumbnails/ directory. MCP tool returns path.

MCP Tool Interface

generate_thumbnail(
  title: "One Person Built OpenClaw",
  subtitle: "Why Solo Founders Win",
  template: "technologia_talks",
  layout: "split_panel" | "centered" | "text_heavy",
  provider: "local" | "flux" | "gemini" | "both"
)

// provider="local" → programmatic renderer (default)
// provider="flux"/"gemini"/"both" → AI generation (existing)
